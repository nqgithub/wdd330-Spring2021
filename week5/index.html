Chapter 10: Testing and Debugging


Errors, Exceptions, and Warnings

  • Errors can be one od the following
      • System error ― there’s a problem with the system or external devices with which the program is interacting.
      Programmer error ― the program contains incorrect syntax or faulty logic; it could even be as simple as a typo.
      User error ― the user has entered data incorrectly, which the program is unable to handle.
  • External errors like monitors, cables, etc. we cann't do much but programming errors are something that as a programmer we are in charge of. User errors are also another possibility. As programmers we should help make programs more user friendly so users don't make errors.

    • Exceptions and Stack Traces

  • Exception: an error that produces a return value that can then be used by the program to deal with the error.
  • Stack Traces: An exception will also produce astack trace. This is a sequence of functions or method calls that lead to the point where the error occurred.

    • Warnings

  • A warning can occur if there’s an error in the code that isn't enough to cause the program to crash. This means the program will continue to run after a warning. This might sound good, but it can be problematic, since the issue that produced the warning may cause the program to continue running incorrectly.
  • Example: Assigning a value to a variable that’s undeclared
  • Runtime error: When a runtime error occurs in the browser, the HTML will still appear, but the JavaScript code will stop working in the background.
  • Warning error: If a warning occurs, the JavaScript will continue to run (although possibly incorrectly).

    • The Importance of Testing and Debugging

  • JavaScript is a fairly forgiving language when it comes to errors.
  • When an error occurs Java will still run and the error will occur silently in the background.
